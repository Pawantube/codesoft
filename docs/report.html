2. Assigned Roles & Responsibilities

  • Implement MERN features for a modern job board application.
  • Build job discovery (home, search, filters) and detailed job view with apply flow.
  • Enable resume uploads and secure file processing for applications.
  • Support employer dashboards and application management endpoints.
  • Integrate real‑time communication (chat, collaborative coding) via Socket.IO.
  • Enhance PWA experience including service worker caching and push notifications.

3. Technologies / Tools Used

Frontend

  • React (Vite) for SPA
  • Routing: react‑router
  • UI: Utility classes (Tailwind‑style)
  • PWA: client/public/manifest.webmanifest, client/public/sw.js

Backend

  • Node.js + Express
  • Auth + Role‑based access (middleware)
  • File uploads: Multer (memory) → Cloudinary
  • Realtime: Socket.IO for chat and live coding

API base configured via VITE_API_URL (default http://localhost:5000). Client API wrapper: client/src/utils/api.js.

5. Challenges Faced & Solutions Implemented

  • Secure file handling. Used Multer memory storage with strict file filters (PDF/DOC/RTF) and image‑only constraints; uploaded via Cloudinary streams to avoid local persistence.
  • Stateful search UX. Synced filters with URL query params to support pagination, deep‑links, and shareable searches.
  • Realtime consistency. Managed Socket.IO room lifecycle and optimistic UI updates with React effects and refs.
  • Offline resilience. Implemented cache versioning and graceful fallbacks for navigation/API in the service worker.
